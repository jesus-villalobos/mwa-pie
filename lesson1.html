Lesson Content:

Types:

In Python, "types" are names that refer to certain kinds of things within the language. For the purposes of these lessons, we will focus on two types:
  • Text Types
  • Numeric Types

Text Types:

The Text Type is encapsulated by the string. Strings are they way that Python developers use plain words within code. Strings carry meaning within them, and are used to interface with the user. A string is text that is found within either single OR double quotes; it does not matter, as long as either side of a string has the same kind. Strings also have some special functions associated with them:
  • To ask Python if something is a string, you can input it into the special type() method.
  • To convert something to a string, you can use the special str() method.
Try these things in the terminal above!!

NOTE: A comment is NOT considered a string. A comment is used only in the script of code, but is never run by the terminal. Comments are identified by one pound symbol, #, or wrapped in triple quotes, """. 

# This is a comment. It will not be run.
# Neither will this one. The symbol comments out one line at a time.

"""This is also a comment"""
"""A pound in here, #, does not matter because it is already commented out."""

"This is a string."
'So is this.'
# ---> Either double or single quotes work.

"A pound in here, #, does nothing because it is being interpreted as plain English."

>>> str(3)
'3'
>>> str(909230934)
'909230934'
>>> type("Hello World")
< class 'str'>           --> 'str' stands for 'string'

Numeric Types:

There are two different kinds of numbers that fall into the numeric type:
  • Integer, int
    • The int is any whole number.
  • Floating Point Number, float
    • The float is ANY number with a decimal point, no matter how many digits come after it.
Python uses instances of the numeric type to perform math calculations, and carry meaning between different parts of programs. ints and floats can be used simultaneously in math operations, but there are certain points where it becomes useful to use one kind or the other.

You can use special instances of Python functions to convert between either type. These are the int() and float() functions.

Numbers also have some special functions associated with them:
  • To ask Python if something is a number (float or int), you can input it into the special type() method.
  • To convert a number to a float or int, you can use the special float() or int() methods, respectively.
Try these things in the terminal above!! 
# NOTE: the three arrows represent Python input, and the line after it is the output.

>>> type(39.07)
< class 'float'>
>>> type(9839483)
< class 'int'>

"""To convert between either type:"""

>>> float(76)  # int --> float
76.0
>>> float(4)   # int --> float
4.0

>>> int(36.97)    # float --> int
36
>>> int(3.141592658597373982308940934)     # float --> int
3

Python Math Operators:

In Python, we use the same ol' math operations used in everyday math, plus 3 addition functions. The full list of operations, and their symbol is:
  • Addition, +
  • Subtraction, -
  • Multiplication, *
  • Division, /
  • Power, **
  • Floor Division, //
  • Modulus, %
  • Equality, ==
The three new operations are Floor Division, Modulus, and Equality. Here is a description of each, along with a few examples.

Floor Division, // is the same as regular division, it simply cuts off everything after the decimal point if there is anything.
Modulus, % returns the remainder of division. You can think about it as "lazy division" because it ignores performing long division into decimal places, and just spits out the remainder.
Equality, == returns whether two things are equal to each other. The output will always be either True or False. 

"""Familiar Operations:"""
>>> 2 + 3
5
>>> 2 - 3
-1
>>> 2 * 3
6
>>> 6 / 3
2.0
>>> 3 ** 2
9

"""New Operations:"""
>>> 6 // 3
2
>>> 13 // 3      --> 13 / 3 = 4.33333333
4
>>> 13 % 3       --> 13 / 3 = 4 with remainder 1
1
>>> 15 % 4       --> 15 / 4 = 3 with remainder 3
3
>>> 3 == 3.0     --> The type doesn't matter; they are equal regardless
True
>>> 567 == 36    --> They are obviously not equal
False
